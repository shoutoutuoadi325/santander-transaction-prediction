## 六、实验严谨性与防泄露说明（新增）

> 对应 PPT Ground Rules：防止数据泄露、合理切分训练/验证、记录随机种子。

### 1. 数据集划分策略

| 模块 | 划分方式 | 原因 |
|------|----------|------|
| 分类主实验 | 5 折 StratifiedKFold，random_state=42 | 结果更稳定，报告对比实验更有说服力 |
| 离群检测、频繁模式、聚类辅助分析 | 固定 80/20 分层划分，random_state=42 | 计算量大，单次划分够用；报告中需说明原因 |
| 消融实验 | 基于 Codex 的 5 折 CV 结果 | 与主实验保持一致 |

### 2. Frequency Encoding 防泄露

frequency encoding 使用训练集 + 测试集合并统计频率（只用数值分布，不涉及 `target`），这是 Kaggle 竞赛中的标准做法。报告中需明确说明：

> frequency encoding 仅利用特征值在 train+test 中的出现次数（无标签信息），不存在目标泄露。在 fold 内统计频率是更严格的做法，本项目选择 train+test 合并统计以与 Kaggle 竞赛实践对齐。

### 3. 所有辅助模型的 fit/transform 规则

- StandardScaler、PCA、聚类模型、Isolation Forest：**只在训练集 fit，再 transform 验证集**
- 不能把验证集数据混入 fit 过程

```python
# 正确做法
scaler.fit(X_train)
X_train_s = scaler.transform(X_train)
X_valid_s = scaler.transform(X_valid)  # 不 fit_transform

# 错误做法（泄露）
scaler.fit_transform(pd.concat([X_train, X_valid]))
```

### 4. 随机种子

所有模型、划分、随机操作统一使用 `random_state=42`，保证实验可复现。

## 七、频繁模式挖掘设计（新增）

> 这是原方案缺失的模块，对应课程评分中"频繁模式"一类，补上后四类方法全覆盖。

### 研究问题

正例客户（`target=1`）中，哪些特征的取值区间倾向于同时出现？这些规则能否解释高交易倾向客户的行为模式？

### 流程

**Step 1：离散化**

对 200 个连续特征做等频分箱（3档：低/中/高），转换为类别标签：

```python
import pandas as pd

def discretize_features(df, feature_cols, n_bins=3, labels=["low", "mid", "high"]):
    df_disc = df[feature_cols].copy()
    for col in feature_cols:
        df_disc[col] = pd.qcut(df[col], q=n_bins, labels=labels, duplicates="drop")
    return df_disc.astype(str)
```

**Step 2：只对正例子集挖掘**

```python
pos_disc = discretize_features(X_train[y_train == 1], feature_cols)
```

**Step 3：构造事务数据库并运行 FP-Growth**

```python
from mlxtend.frequent_patterns import fpgrowth, association_rules
from mlxtend.preprocessing import TransactionEncoder

# 每行转为 item 列表，格式如 ["var_0=high", "var_5=low", ...]
transactions = []
for _, row in pos_disc.iterrows():
    items = [f"{col}={val}" for col, val in row.items()]
    transactions.append(items)

te = TransactionEncoder()
te_array = te.fit_transform(transactions)
df_te = pd.DataFrame(te_array, columns=te.columns_)

frequent_items = fpgrowth(df_te, min_support=0.3, use_colnames=True)
rules = association_rules(frequent_items, metric="lift", min_threshold=1.2)
rules_sorted = rules.sort_values("lift", ascending=False).head(20)
```

**Step 4：输出图表**

- 支持度最高的 Top 20 频繁项集表格
- Lift 最高的 Top 20 关联规则表格
- 关联规则散点图（x: support, y: confidence, 颜色: lift）

**Step 5：报告中的结论示例**

> 在正例客户群体中，发现 `var_X=high` 与 `var_Y=low` 共现支持度为 0.35，Lift 为 1.6，表明两者具有显著的正向关联，可能反映了特定的交易行为模式。

## 十一、频率编码与特征增强

虽然 Santander 是数值特征，但可以做一个简单有效的特征工程：frequency encoding。

### 思路

对每个特征 `var_i`，统计某个取值在训练集和测试集中出现的频率。因为数值特征可能有重复值，重复值频率本身可能携带分布信息。

```python
def add_frequency_features(train, test, feature_cols):
    full = pd.concat([train[feature_cols], test[feature_cols]], axis=0)
    for col in feature_cols:
        freq = full[col].value_counts()
        train[col + "_freq"] = train[col].map(freq)
        test[col + "_freq"] = test[col].map(freq)
    return train, test
```

然后比较：

### 评估指标

- **主指标**：ROC-AUC

为什么不用 Accuracy 作为主指标：

> 因为正负样本不平衡，模型即使偏向预测多数类，也可能得到较高 Accuracy，但这并不能说明模型真正识别出了交易倾向客户。

import pandas as pd
from sklearn.preprocessing import StandardScaler

def add_frequency_features(train, test, feature_cols):
    full = pd.concat([train[feature_cols], test[feature_cols]], axis=0)
    for col in feature_cols:
        freq = full[col].value_counts()
        train[col + "_freq"] = train[col].map(freq)
        test[col + "_freq"] = test[col].map(freq)
    return train, test

def standardize(X_train, X_valid, X_test=None):

### `frequent_patterns.py`（新增）

```python
import pandas as pd
from mlxtend.frequent_patterns import fpgrowth, association_rules
from mlxtend.preprocessing import TransactionEncoder

def discretize_features(df, feature_cols, n_bins=3, labels=["low", "mid", "high"]):
    df_disc = df[feature_cols].copy()
    for col in feature_cols:
        df_disc[col] = pd.qcut(df[col], q=n_bins, labels=labels, duplicates="drop")
    return df_disc.astype(str)

def mine_frequent_patterns(X_pos, feature_cols, min_support=0.3, min_lift=1.2):
    disc = discretize_features(X_pos, feature_cols)
    transactions = [[f"{col}={val}" for col, val in row.items()] for _, row in disc.iterrows()]
    te = TransactionEncoder()
    df_te = pd.DataFrame(te.fit_transform(transactions), columns=te.columns_)
    frequent = fpgrowth(df_te, min_support=min_support, use_colnames=True)
    rules = association_rules(frequent, metric="lift", min_threshold=min_lift)
    return frequent, rules.sort_values("lift", ascending=False)
```
